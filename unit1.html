Basic Terminology

Basic Data Units

  • Data: Raw values or sets of values. Example: 24, John Smith, 01/01/2000.
  • Data Item: A single unit of value. Example: A phone number or an address string.
  • Group Items: Data items that can be divided into smaller sub-items. Example: Full Name can be split into First Name and Last Name.
  • Elementary Items: Data items that cannot be divided any further. Example: Employee ID (e.g., E101).

Entities and Properties

  • Entity & Attribute: An entity is an object, and attributes are its specific properties. Example: A Car is the entity; its Color, Model, and Price are the attributes.
  • Entity Set: A collection of similar entities. Example: A group of all cars in a dealership inventory.

Storage Hierarchy

  • Field: A single unit of information representing one attribute. Example: A single cell in a spreadsheet containing the value Red.
  • Record: A complete collection of fields belonging to one specific entity. Example: A single row in a spreadsheet containing one car's ID, Model, and Color.
  • File: A master collection of all related records within an entity set. Example: The entire spreadsheet or database table named Inventory_List.

Classification of Data Structures

Data structures can broadly be classified into two main categories:

1. Primitive Data Structures

Primitive data structures are the most basic types of data structures provided by programming languages. They include:

  • Integer: Whole numbers (e.g., 1, 2, 100).
  • Float: Numbers with decimal points (e.g., 3.14, 0.001).
  • Character: Single characters (e.g., ‘A’, ‘B’).
  • Boolean: Logical values (true or false).

Sample Program in C++ (Primitive Data Types)

#include <iostream>
using namespace std;

int main() {
    int num = 10;        // Integer
    float pi = 3.14;     // Float
    char letter = 'A';   // Character
    bool isHappy = true; // Boolean

    cout << "Integer: " << num << endl;
    cout << "Float: " << pi << endl;
    cout << "Character: " << letter << endl;
    cout << "Boolean: " << isHappy << endl;
    
    return 0;
}

2. Non-Primitive Data Structures

Non-primitive data structures are more complex and are built using primitive data types. They are further divided into:

a. Linear Data Structures

In linear data structures, elements are arranged sequentially, and each element has a unique predecessor and successor (except the first and last elements).

  • Array: A collection of elements of the same type, stored in contiguous memory locations.
  • Linked List: A collection of nodes where each node contains data and a pointer to the next node.
  • Stack: A collection of elements following the Last In First Out (LIFO) principle.
  • Queue: A collection of elements following the First In First Out (FIFO) principle.

Sample Program in C++ (Array)

#include <iostream>
using namespace std;

int main() {
    int arr[5] = {1, 2, 3, 4, 5}; // Array of integers
    
    cout << "Array elements:" << endl;
    for (int i = 0; i < 5; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
    
    return 0;
}

b. Non-Linear Data Structures

Elements are arranged in a hierarchical or interconnected structure rather than sequentially.

  • Tree: A hierarchical layout where nodes connect to parent and child nodes.
  • Graph: A network of nodes (vertices) connected by paths (edges) to show relationships.

Sample Program in C++ (Basic Tree Traversal)

#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* left;
    Node* right;
    Node(int val) {
        data = val;
        left = right = nullptr;
    }
};

void inOrder(Node* root) {
    if (root == nullptr) return;
    inOrder(root->left);
    cout << root->data << " ";
    inOrder(root->right);
}

int main() {
    Node* root = new Node(1);
    root->left = new Node(2);
    root->right = new Node(3);
    root->left->left = new Node(4);
    root->left->right = new Node(5);

    cout << "In-order Traversal: ";
    inOrder(root);
    cout << endl;
    return 0;
}

c. File Structures

Specialized structures designed for storing, retrieving, and updating data efficiently in secondary storage (e.g., B-trees, hash files).

Applications of Data Structures

  • Arrays: Implementing stacks, queues, matrices, tables, and storing uniform datasets.
  • Linked Lists: Dynamic data storage, implementing stacks, queues, and handling hash table chaining.
  • Trees: Managing hierarchical layouts like file systems, organization charts, networks, and search dictionaries.
  • Graphs: Modeling network connections (social, transport, computer) and executing path-finding routing algorithms.
  • Hash Tables: Quick key-value lookups for associative arrays, caching, and database indexing.
  • Stacks: Handling undo/redo logic, processing function calls, and parsing mathematical expressions.
  • Queues: Managing print buffers, processing background task lines, and handling server job scheduling.
