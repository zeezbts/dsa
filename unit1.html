Basic Terminology

Basic Data Units

  • Data: Raw values or sets of values. Example: 24, John Smith, 01/01/2000.
  • Data Item: A single unit of value. Example: A phone number or an address string.
  • Group Items: Data items that can be divided into smaller sub-items. Example: Full Name can be split into First Name and Last Name.
  • Elementary Items: Data items that cannot be divided any further. Example: Employee ID (e.g., E101).

Entities and Properties

  • Entity & Attribute: An entity is an object, and attributes are its specific properties. Example: A Car is the entity; its Color, Model, and Price are the attributes.
  • Entity Set: A collection of similar entities. Example: A group of all cars in a dealership inventory.

Storage Hierarchy

  • Field: A single unit of information representing one attribute. Example: A single cell in a spreadsheet containing the value Red.
  • Record: A complete collection of fields belonging to one specific entity. Example: A single row in a spreadsheet containing one car's ID, Model, and Color.
  • File: A master collection of all related records within an entity set. Example: The entire spreadsheet or database table named Inventory_List.

Classification of Data Structures

Data structures can broadly be classified into two main categories:

1. Primitive Data Structures

Primitive data structures are the most basic types of data structures provided by programming languages. They include:

  • Integer: Whole numbers (e.g., 1, 2, 100).
  • Float: Numbers with decimal points (e.g., 3.14, 0.001).
  • Character: Single characters (e.g., ‘A’, ‘B’).
  • Boolean: Logical values (true or false).

Sample Program in C++ (Primitive Data Types)

#include <iostream>
using namespace std;

int main() {
    int num = 10;        // Integer
    float pi = 3.14;     // Float
    char letter = 'A';   // Character
    bool isHappy = true; // Boolean

    cout << "Integer: " << num << endl;
    cout << "Float: " << pi << endl;
    cout << "Character: " << letter << endl;
    cout << "Boolean: " << isHappy << endl;
    
    return 0;
}

2. Non-Primitive Data Structures

Non-primitive data structures are more complex and are built using primitive data types. They are further divided into:

a. Linear Data Structures

In linear data structures, elements are arranged sequentially, and each element has a unique predecessor and successor (except the first and last elements).

  • Array: A collection of elements of the same type, stored in contiguous memory locations.
  • Linked List: A collection of nodes where each node contains data and a pointer to the next node.
  • Stack: A collection of elements following the Last In First Out (LIFO) principle.
  • Queue: A collection of elements following the First In First Out (FIFO) principle.

Sample Program in C++ (Array)

#include <iostream>
using namespace std;

int main() {
    int arr[5] = {1, 2, 3, 4, 5}; // Array of integers
    
    cout << "Array elements:" << endl;
    for (int i = 0; i < 5; i++) {
        cout << arr[i] << " ";
    }
    cout << endl;
    
    return 0;
}

b. Non-Linear Data Structures

Elements are arranged in a hierarchical or interconnected structure rather than sequentially.

  • Tree: A hierarchical layout where nodes connect to parent and child nodes.
  • Graph: A network of nodes (vertices) connected by paths (edges) to show relationships.

Sample Program in C++ (Basic Tree Traversal)

#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* left;
    Node* right;
    Node(int val) {
        data = val;
        left = right = nullptr;
    }
};

void inOrder(Node* root) {
    if (root == nullptr) return;
    inOrder(root->left);
    cout << root->data << " ";
    inOrder(root->right);
}

int main() {
    Node* root = new Node(1);
    root->left = new Node(2);
    root->right = new Node(3);
    root->left->left = new Node(4);
    root->left->right = new Node(5);

    cout << "In-order Traversal: ";
    inOrder(root);
    cout << endl;
    return 0;
}

c. File Structures

Specialized structures designed for storing, retrieving, and updating data efficiently in secondary storage (e.g., B-trees, hash files).

Applications of Data Structures

  • Arrays: Implementing stacks, queues, matrices, tables, and storing uniform datasets.
  • Linked Lists: Dynamic data storage, implementing stacks, queues, and handling hash table chaining.
  • Trees: Managing hierarchical layouts like file systems, organization charts, networks, and search dictionaries.
  • Graphs: Modeling network connections (social, transport, computer) and executing path-finding routing algorithms.
  • Hash Tables: Quick key-value lookups for associative arrays, caching, and database indexing.
  • Stacks: Handling undo/redo logic, processing function calls, and parsing mathematical expressions.
  • Queues: Managing print buffers, processing background task lines, and handling server job scheduling.

1D vs 2D Array Differences

# Feature 1D Array (One-Dimensional) 2D Array (Two-Dimensional)
1 Definition A linear collection of elements arranged in a single row or line. An array of arrays arranged in a grid structure of rows and columns.
2 Dimensions Has only one dimension (length/size). Has two dimensions (rows and columns).
3 Indexing Requires a single index to access an element: array[i]. Requires two indices to access an element: array[i][j].
4 Declaration Syntax (C/C++) dataType arrayName[size]; dataType arrayName[rows][columns];
5 Visual Analogy A single row of lockers or a simple shopping list. A chessboard, a spreadsheet table, or a digital image grid.
6 Memory Allocation Elements are stored in a continuous, straight linear sequence in RAM. Elements are stored in RAM by flattening the grid using row-major or column-major order.
7 Address Calculation Requires only the index to find the location:
Base + (i * size)		 Requires row index, column index, and total column count:
Base + ((i * Total_Cols) + j) * size
8 Total Elements Formula Equals the declared size of the array. Equals the total number of rows multiplied by columns (Rows * Columns).
9 Initialization Layout Initialized using a single set of braces:
{10, 20, 30}		 Initialized using nested sets of braces:
{{1, 2}, {3, 4}}
10 Primary Use Cases Storing simple linear datasets like names, scores, or time-series data. Storing matrices, coordinate systems, pixel data for images, or game boards.

1D Array Example

Stores a simple list of test scores.

// Declaration and initialization
int scores[3] = {85, 92, 78};

// Accessing an element (index 1)
int secondScore = scores[1]; // Value is 92

2D Array Example

Stores a 2x3 seating grid for a small venue.

// Declaration and initialization (2 rows, 3 columns)
int seats[2][3] = {
    {1, 2, 3},  // Row 0
    {4, 5, 6}   // Row 1
};

// Accessing an element (Row 1, Column 2)
int selectedSeat = seats[1][2]; // Value is 6

Applications of Arrays

  • Storing Sequential Data: Saving basic sequential lists like user contacts, test scores, or financial transactions.
  • Matrix Mathematics: Performing mathematical matrix operations, image transformation grids, or coordinate layouts.
  • Building Block for Data Structures: Serving as the foundation to build complex structures like Stacks, Queues, and Hash Tables.
  • Lookup Tables: Fast static value lookups, such as character conversion codes or math constant sets.
  • CPU Scheduling Algorithms: Managing processing sequences in computer systems via simple lookup or priority indexes.
  • Game Board Layouts: Organizing structured grids for games like chessboards, tic-tac-toe, or pixel maps.

1. Big-O Notation (O) — Worst Case / Upper Bound

  • Definition: Represents the absolute maximum time or space an algorithm will ever take.
  • Guarantees: The execution time will never cross or violate this upper bound ceiling.
  • Simplification: Focuses exclusively on the fastest-growing term while dropping lower terms and constants.
  • Real-world Analogy: Booking a flight and being told "it will take at most 5 hours to arrive."
  • Use Case: Used by engineering teams to predict worst-case server loads and performance failures.
// Searching for a value that is either at the very end or not in the array
for (int i = 0; i < n; i++) {
    if (arr[i] == target) return i; 
}
// Time Complexity: O(n) — Checked all 'n' elements in the worst case.

2. Big-Omega Notation (Ω) — Best Case / Lower Bound

  • Definition: Represents the absolute minimum time or space an algorithm requires.
  • Guarantees: The algorithm will never run faster or take fewer steps than this baseline floor.
  • Focus: Examines performance under the most ideal, perfect input conditions.
  • Real-world Analogy: A mechanic telling you, "Fixing this car will cost at least \$50."
  • Use Case: Used to prove that a certain problem cannot be solved faster than a mathematical floor limit.
// Finding the target item immediately on the very first look
for (int i = 0; i < n; i++) {
    if (arr[i] == target) return i; // Target is at index 0
}
// Time Complexity: Ω(1) — Found instantly in a single step.

3. Big-Theta Notation (Θ) — Average / Tight Bound

  • Definition: Represents the precise, tight bound where the best and worst cases scale at the same rate.
  • Guarantees: Sandwiches performance tightly from both above and below.
  • Condition: Applies only if an algorithm's Big-O and Big-Omega expressions are identical.
  • Real-world Analogy: A train journey that takes exactly 45 minutes every single day, without fail.
  • Use Case: Used to describe stable algorithms whose execution loops depend purely on input size, not input values.
// Printing every single item in an array no matter what data is inside it
for (int i = 0; i < n; i++) {
    std::cout << arr[i] << " ";
}
// Time Complexity: Θ(n) — Must loop exactly 'n' times in all circumstances.
