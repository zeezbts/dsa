Basic Terminology in Linked Lists

Core Elements

  • Node: The basic building block containing data and a link pointer.
  • Data: The actual value or payload stored inside the node.
  • Link / Next: The pointer storing the memory address of the next node.

Positions & Pointers

  • Head: The starting pointer pointing to the first node of the list.
  • Tail: The last node of the list, whose next pointer usually equals NULL.
  • NULL: A marker indicating an empty address, signaling the end of the list.

Key Terms & States

  • Traversal: Passing through each node sequentially from start to finish.
  • Overflow: An error state when memory is full and no new node can be added.
  • Underflow: An error state when trying to delete an item from an empty list.

Linked Lists

A linked list is a linear data structure where elements (nodes) are not stored in contiguous memory locations. Instead, each node points to the next using a memory address link.

There are four primary types of linked lists based on how the nodes connect to each other:

1. Singly Linked List

  • Structure: Each node contains data and one forward pointer (next).
  • Direction: Supports forward traversal only.
  • Termination: The final node's pointer links directly to NULL.
  • Memory: Most memory-efficient type as it stores only one pointer per node.
  • Use Case: Implementing basic stacks or simple file allocation tables.

Common Time Complexities

  • Access / Search: O(N) — Must start at the head and walk step-by-step.
  • Insertion at Head: O(1) — Instant step changing the pointer of a new element.
  • Insertion at Tail: O(N) — Requires traversing to the end of the list first (unless a tail pointer is maintained).
  • Deletion at Head: O(1) — Instant adjustment moving the head pointer forward by one.

Implementation Example

struct Node {
    int data;
    Node* next;
};
// Node initialization
Node* first = new Node{10, nullptr};
Node* second = new Node{20, nullptr};
first->next = second; // 10 -> 20 -> NULL

Consolidated Singly Linked List Operations

#include <iostream>

struct Node {
    int data;
    Node* next;
};

// 1. Insertion at Head - O(1)
void insertAtHead(Node*& head, int val) {
    head = new Node{val, head};
}

// 2. Deletion at Head - O(1)
void deleteAtHead(Node*& head) {
    if (!head) return;
    Node* temp = head;
    head = head->next;
    delete temp;
}

// 3. Insertion at Tail - O(N)
void insertAtTail(Node*& head, int val) {
    Node* newNode = new Node{val, nullptr};
    if (!head) { head = newNode; return; }
    Node* temp = head;
    while (temp->next) temp = temp->next;
    temp->next = newNode;
}

// 4. Access / Search - O(N)
bool search(Node* head, int target) {
    while (head) {
        if (head->data == target) return true;
        head = head->next;
    }
    return false;
}

void printList(Node* head) {
    while (head) {
        std::cout << head->data << " -> ";
        head = head->next;
    }
    std::cout << "NULL\n";
}

int main() {
    Node* head = nullptr;

    insertAtHead(head, 20);
    insertAtHead(head, 10);
    insertAtTail(head, 30);
    printList(head); // Output: 10 -> 20 -> 30 -> NULL

    std::cout << "Found 20? " << (search(head, 20) ? "Yes" : "No") << "\n";

    deleteAtHead(head);
    printList(head); // Output: 20 -> 30 -> NULL

    return 0;
}

Singly Linked List

2. Doubly Linked List

  • Structure: Each node contains data, a forward pointer (next), and a backward pointer (prev).
  • Direction: Supports full bidirectional traversal (forward and backward).
  • Termination: Both the first node's prev and last node's next point to NULL.
  • Memory: Requires extra memory allocation to store two pointers per node.
  • Use Case: Web browser back/forward history or undo/redo application systems.

Time Complexities & Core Examples

  • Search / Access — O(N): Must check items sequentially (e.g., locating value 25 inside a list of files).
  • Insert / Delete at Ends — O(1): Instant manipulation if head/tail references exist (e.g., adding a song to the end of a playlist).
  • Insert / Delete at Given Node — O(1): Instant restructuring if you have a direct pointer to that node (e.g., removing the currently selected text state).

Implementation Example

struct Node {
    int data;
    Node* next;
    Node* prev;
};
// Node initialization
Node* first = new Node{10, nullptr, nullptr};
Node* second = new Node{20, nullptr, first};
first->next = second; // NULL <- 10 <=> 20 -> NULL

C++ Code Implementation

struct Node {
    int data;
    Node* next;
    Node* prev;
};

// Insertion at Head - O(1)
void insertAtHead(Node*& head, Node*& tail, int val) {
    Node* newNode = new Node{val, head, nullptr};
    if (head != nullptr) head->prev = newNode;
    else tail = newNode;
    head = newNode;
}

// Deletion at Head - O(1)
void deleteAtHead(Node*& head, Node*& tail) {
    if (head == nullptr) return;
    Node* temp = head;
    head = head->next;
    if (head != nullptr) head->prev = nullptr;
    else tail = nullptr;
    delete temp;
}

Doubly Linked List

3. Circular Singly Linked List

  • Structure: A singly linked list where the final node connects back to the first node.
  • Direction: Unidirectional forward navigation that loops continuously.
  • Termination: No node ever points to NULL.
  • Memory: Uses the same minimal pointer memory space as a standard singly linked list.
  • Use Case: Round-robin CPU time-sharing scheduling systems or audio playlist looping.

Time Complexities & Core Examples

  • Search / Access — O(N): Must scan sequentially. The search terminates when the pointer cycles back to the starting node (e.g., verifying an active task in an processing loop).
  • Insert / Delete at Head — O(1) or O(N): Runs in O(1) time if you track a reference to the tail node directly, as you can bypass full traversal to update the wrap-around link.
  • Insert at Tail — O(1) or O(N): Takes O(1) if a tail pointer is actively maintained; otherwise, it requires walking the list to find the element that wraps back around.

Implementation Example

struct Node {
    int data;
    Node* next;
};
// Node initialization
Node* first = new Node{10, nullptr};
Node* second = new Node{20, first}; // Loops back to first
first->next = second;               // 10 -> 20 -> (loops back to 10)

Consolidated C++ Program

#include <iostream>

struct Node {
    int data;
    Node* next;
};

// 1. Insertion at Head - O(1) using tail pointer tracking
void insertAtHead(Node*& tail, int val) {
    Node* newNode = new Node{val, nullptr};
    if (!tail) {
        newNode->next = newNode;
        tail = newNode;
        return;
    }
    newNode->next = tail->next; // Point new node to the old head
    tail->next = newNode;       // Link tail to the new head
}

// 2. Insertion at Tail - O(1) using tail pointer tracking
void insertAtTail(Node*& tail, int val) {
    Node* newNode = new Node{val, nullptr};
    if (!tail) {
        newNode->next = newNode;
        tail = newNode;
        return;
    }
    newNode->next = tail->next; // Point new node to the head
    tail->next = newNode;       // Link old tail to new node
    tail = newNode;             // Update tail pointer marker
}

// 3. Deletion at Head - O(1)
void deleteAtHead(Node*& tail) {
    if (!tail) return; // Underflow guard
    Node* head = tail->next;
    if (head == tail) { // Only one node in the list
        delete head;
        tail = nullptr;
        return;
    }
    tail->next = head->next; // Link tail to the second node
    delete head;             // Clear old head memory
}

// 4. Access / Search - O(N)
bool search(Node* tail, int target) {
    if (!tail) return false;
    Node* current = tail->next; // Start from head
    do {
        if (current->data == target) return true;
        current = current->next;
    } while (current != tail->next);
    return false;
}

void printList(Node* tail) {
    if (!tail) { std::cout << "Empty List\n"; return; }
    Node* current = tail->next; // Start from head
    do {
        std::cout << current->data << " -> ";
        current = current->next;
    } while (current != tail->next);
    std::cout << "(Loops back to " << tail->next->data << ")\n";
}

int main() {
    Node* tail = nullptr; // Track list via tail instead of head for efficiency

    insertAtHead(tail, 20);
    insertAtHead(tail, 10);
    insertAtTail(tail, 30);
    printList(tail); // Output: 10 -> 20 -> 30 -> (Loops back to 10)

    std::cout << "Found 20? " << (search(tail, 20) ? "Yes" : "No") << "\n";

    deleteAtHead(tail);
    printList(tail); // Output: 20 -> 30 -> (Loops back to 20)

    return 0;
}

4. Circular Doubly Linked List

  • Structure: A doubly linked list where end nodes cross-connect directly with each other.
  • Direction: Continuous loop traversal available in both directions.
  • Termination: No termination point exists; pointers never equal NULL.
  • Memory: Most complex variant requiring maximum pointer storage overhead.
  • Use Case: Infinite game item carousel selectors or browser tab shuffling layouts.

Time Complexities & Core Examples

  • Search / Access — O(N): Must scan sequentially forward or backward. You break the traversal loop when your cursor pointer meets the starting node address again.
  • Insert / Delete at Ends — O(1): Runs in perfect constant time if you hold a simple head pointer tracker. Bypasses list traversal since head->prev gives you direct access to the tail node.
  • Insert / Delete at Given Node — O(1): Performs instant pointer shifting because any isolated node explicitly stores the location details of its neighboring nodes.

Implementation Example

struct Node {
    int data;
    Node* next;
    Node* prev;
};
// Node initialization
Node* first = new Node{10, nullptr, nullptr};
Node* second = new Node{20, first, first}; // next/prev link to first
first->next = second;
first->prev = second;                      // first prev links to second

Consolidated C++ Program

#include <iostream>

struct Node {
    int data;
    Node* next;
    Node* prev;
};

// 1. Insertion at Head - O(1)
void insertAtHead(Node*& head, int val) {
    Node* newNode = new Node{val, nullptr, nullptr};
    if (!head) {
        newNode->next = newNode;
        newNode->prev = newNode;
        head = newNode;
        return;
    }
    Node* tail = head->prev; // Direct access to tail
    
    newNode->next = head;
    newNode->prev = tail;
    head->prev = newNode;
    tail->next = newNode;
    head = newNode;
}

// 2. Insertion at Tail - O(1)
void insertAtTail(Node*& head, int val) {
    Node* newNode = new Node{val, nullptr, nullptr};
    if (!head) {
        newNode->next = newNode;
        newNode->prev = newNode;
        head = newNode;
        return;
    }
    Node* tail = head->prev; // Direct access to tail
    
    newNode->next = head;
    newNode->prev = tail;
    tail->next = newNode;
    head->prev = newNode;
}

// 3. Deletion at Head - O(1)
void deleteAtHead(Node*& head) {
    if (!head) return; // Underflow guard
    Node* tail = head->prev;
    Node* temp = head;
    
    if (head == tail) { // Only one node in the list
        delete head;
        head = nullptr;
        return;
    }
    head = head->next;
    head->prev = tail;
    tail->next = head;
    delete temp;
}

void printForward(Node* head) {
    if (!head) { std::cout << "Empty List\n"; return; }
    Node* current = head;
    do {
        std::cout << current->data << " <=> ";
        current = current->next;
    } while (current != head);
    std::cout << "(Loops back to " << head->data << ")\n";
}

int main() {
    Node* head = nullptr;

    insertAtHead(head, 20);
    insertAtHead(head, 10);
    insertAtTail(head, 30);
    printForward(head); // Output: 10 <=> 20 <=> 30 <=> (Loops back to 10)

    deleteAtHead(head);
    printForward(head); // Output: 20 <=> 30 <=> (Loops back to 20)

    return 0;
}

Applications of Linked Lists

  • Browser History: Managing back and forward page routing via Doubly Linked Lists.
  • Music Playlists: Sequential song queues with fluent track-to-track navigation.
  • Undo/Redo Tracking: Stepping back and forth through chronological state changes.
  • Image Viewers: Cycling next and previous photo assets cleanly inside galleries.
  • OS CPU Scheduling: Cycling active threads using round-robin Circular Linked Lists.
  • RAM Allocation: Managing available and active blocks inside system heap records.
  • Data Structure Engine: Acting as the dynamic foundation for Stacks and Queues.
  • Hash Chaining: Resolving indexing conflicts cleanly inside crowded Hash Tables.
  • Graph Modeling: Representing connection networks via structural Adjacency Lists.
  • Polynomial Math: Storing varying mathematical terms cleanly as separate dynamic nodes.
