Understanding the Stack Data Structure

A Stack is a linear data structure that follows the Last-In, First-Out (LIFO) principle. This means that the element added last to the stack is the first one to be removed, similar to a physical stack of plates.

Introduction to Stacks

In a stack, all insertion and deletion operations take place at a single end called the Top. It is an abstract data type (ADT) commonly used across computer architecture and algorithms to manage temporary data.

  • LIFO Structure: Last element inserted is the first element accessed.
  • Dynamic/Static: Can be fixed size or dynamic depending on the design.
  • Single Pointer: Tracked entirely via a single reference variable called top.

Operations on a Stack

Whether implemented via arrays or linked lists, standard stacks support these core operations:

  • Push: Adds an element to the top of the stack.
  • Pop: Removes and returns the top element from the stack.
  • Peek/Top: Returns the top element without removing it from the stack.
  • isEmpty: Checks if the stack contains zero elements.
  • isFull: Checks if a fixed-size stack has reached its maximum capacity.

Array Representation of Stacks

An array representation uses a sequential, continuous memory block of fixed size to store data. A counter variable top tracks the index of the highest element.

Algorithmic Implementations (Array)

Push Operation:

  1. Check for Stack Overflow: If top == MAX_SIZE - 1, the stack is full; abort.
  2. Increment the pointer: top = top + 1.
  3. Insert data: Stack[top] = value.

Pop Operation:

  1. Check for Stack Underflow: If top == -1, the stack is empty; abort.
  2. Fetch data: value = Stack[top].
  3. Decrement the pointer: top = top - 1.
  4. Return value.

Linked Representation of Stacks

A linked representation uses dynamic nodes to build the stack. Each node contains a data field and a pointer field linking to the next node below it. The pointer variable top acts as the head of the list, pointing directly to the topmost node.

  • No Size Limit: Grows dynamically during execution without standard overflow errors.
  • Extra Overhead: Requires extra memory for pointer storage in each node.

Operations on a Linked Stack

In a linked stack, items are inserted and removed from the front of the single linked list to maintain O(1) constant time complexity.

Algorithmic Implementations (Linked List)

Push Operation (Linked):

  1. Allocate a new node: newNode = allocate Node.
  2. Set data: newNode.data = value.
  3. Link new node to current stack: newNode.next = top.
  4. Update top pointer: top = newNode.

Pop Operation (Linked):

  1. Check for Stack Underflow: If top == NULL, the stack is empty; abort.
  2. Target top node: tempNode = top.
  3. Advance top pointer: top = top.next.
  4. Extract data: value = tempNode.data.
  5. Free memory: free tempNode.
  6. Return value.

Applications of Stacks

Stacks play a vital role in core operating system and software engine functions:

  • Expression Evaluation & Parsing: Used by compilers to evaluate prefix, infix, and postfix notations (Polish notations).
  • Syntax Checking: Verifies balancing pairs of parentheses, braces, and brackets in source code text.
  • Function Call Management: Stores return addresses and local parameters during nested or recursive functions.
  • Undo/Redo Mechanisms: Powers back-tracking and historic recovery states inside text editors and web browsers.

Comparison: Array vs. Linked Representation

Feature Array Representation Linked Representation
Size Fixed size defined at declaration Dynamic size determined at runtime
Memory Allocation Static / Compile-time allocation Dynamic / Heap allocation at runtime
Extra Memory Pointer No pointer memory required per item Requires memory for an address pointer per node
Errors Prone to Stack Overflow Prone to Heap Exhaustion (Out of Memory)
Access Speed Faster due to memory locality Slightly slower due to pointer traversal
Introduction to Queues - Data Structures

Understanding the Queue Data Structure

A Queue is a linear data structure that operates on the First-In, First-Out (FIFO) principle. This means that the first element added to the queue will be the very first one to be removed, mimicking a real-world waiting line.

1. Introduction to Queues

In a queue, data manipulation happens at two distinct boundaries rather than a single end:

  • Rear (Tail): The point where new elements are inserted into the structure.
  • Front (Head): The point where existing elements are accessed and deleted.

2. Core Operations on a Queue

Regardless of its underlying code implementation, every standard queue relies on these fundamental operations:

  • Enqueue: Adds an element to the rear of the queue.
  • Dequeue: Removes and returns the element located at the front of the queue.
  • Peek/Front: Inspects the front element without altering the data structure.
  • isEmpty: Evaluates whether the queue contains zero elements.
  • isFull: Validates if a fixed-capacity array implementation has maxed out its storage space.

3. Array Representation of Queues

An array implementation allocates a continuous, fixed-size memory sequence. It tracks positions using two independent pointer index markers: front and rear.

Algorithmic Implementations (Array)

Enqueue Operation:

  1. Check for Queue Overflow: If rear == MAX_SIZE - 1, the queue is full; abort.
  2. Handle initial element insertion: If front == -1, update front = 0.
  3. Advance the tail: rear = rear + 1.
  4. Store value: Queue[rear] = value.

Dequeue Operation:

  1. Check for Queue Underflow: If front == -1 or front > rear, the queue is empty; abort.
  2. Retrieve value: value = Queue[front].
  3. Advance the head: front = front + 1.
  4. Return value.

4. Linked Representation of Queues

A linked queue leverages dynamic node structures connected sequentially by memory address references. The front pointer targets the first node in the list, while the rear pointer targets the final node.

Algorithmic Implementations (Linked List)

Enqueue Operation (Linked):

  1. Allocate memory: newNode = allocate Node.
  2. Populate data fields: newNode.data = value and newNode.next = NULL.
  3. If rear == NULL, initialize queue: front = rear = newNode.
  4. Else, link to trailing edge: rear.next = newNode followed by rear = newNode.

Dequeue Operation (Linked):

  1. Check for Queue Underflow: If front == NULL, the queue is empty; abort.
  2. Hold target node reference: tempNode = front.
  3. Update head location: front = front.next.
  4. If front == NULL, clean tail reference: rear = NULL.
  5. Isolate data payload: value = tempNode.data.
  6. Deallocate memory block: free tempNode.
  7. Return value.

5. Types of Queues

Beyond the basic linear structure, queues are modified to optimize data flow patterns:

  • Simple Queue: Insertion occurs strictly at the rear; deletion occurs strictly at the front.
  • Circular Queue: The last position connects back to the first position (rear = (rear + 1) % MAX_SIZE) to prevent unusable dead space.
  • Double-Ended Queue (Deque): Allows insertion and deletion operations seamlessly at both front and rear boundaries.
  • Priority Queue: Elements are assigned a priority value. Higher priority elements are dequeued before lower-priority elements.

6. Applications of Queues

Queues are utilized across distributed networks, operating systems, and asynchronous system pipelines:

  • CPU & Disk Scheduling: Manages task assignment orders using First-Come-First-Served policies.
  • Asynchronous Data Buffering: Regulates I/O communication pipelines like print spools or video stream data packets.
  • Network Packet Management: Buffers streaming web packets arriving at routers to prevent data drop spikes.
  • Breadth-First Search (BFS): Acts as the foundational data tracking layer for traversing graph structures level by level.
