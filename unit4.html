A Tree is a non-linear, hierarchical data structure composed of nodes connected by edges. Unlike sequential arrays, stacks, or queues, trees organize data in parent-child relationships.

1. Introduction to Trees

To understand trees, it is essential to look at their structural components:

  • Root: The topmost node of a tree with no parent node.
  • Edge: The connecting link or structural line between a parent and child node.
  • Leaf Node: An external node that does not have any child nodes.
  • Internal Node: A node that contains at least one child node underneath it.
  • Height: The total number of edges on the longest path from that node down to a leaf node.

2. Types of Trees

Trees are classified based on architectural constraints and value ordering rules:

  • General Tree: A tree structure where each node can have an infinite number of child nodes.
  • Binary Tree: A structured tree where every parent node can have a maximum of two children.
  • Binary Search Tree (BST): An ordered binary tree where left child nodes are smaller, and right child nodes are larger than the parent.
  • AVL Tree: A self-balancing search tree where sub-tree height differences never exceed one.

3. Creating a Binary Tree from a General Tree

A general tree can be systematically mapped into a clean binary tree layout using the Left-Child, Right-Sibling (LCRS) algorithm:

  1. Identify Left Child: Connect the leftmost direct child node as the absolute Left Child in the binary tree map.
  2. Identify Sibling: Connect the immediate right sibling of that child node as its new Right Child.
  3. Isolate: Wipe out all other old structural paths to leave a clean dual-branch configuration.

4. Traversing a Binary Tree

Traversing means visiting every single node in a tree exactly once. This is handled using Depth-First Search (DFS) or Level-Order (BFS) paradigms.

Depth-First Search (DFS) Options

  • Inorder (Left, Root, Right): Recursively runs left, prints the root, then evaluates right branches.
  • Preorder (Root, Left, Right): Evaluates the root immediately before drilling down left and right subtrees.
  • Postorder (Left, Right, Root): Completely evaluates child leaves first before printing their respective parent root.

5. Huffman's Tree

A Huffman Tree is an extended binary tree structure built explicitly to execute data compression algorithms securely.

  • Optimal Prefix Tree: Ensures no symbol sequence code matches the starting prefix of another character code.
  • Frequency Sorting: Assigns short bits to recurring characters and deep paths to rare items to minimize storage footprints.

6. Applications of Trees

Trees act as foundational layers across software compilers, database structures, and runtime systems:

  • File Systems: Powers directory folder mappings across computer operating system architectures.
  • Database Indexing: Employs balanced B/B+ trees to look up data rows inside high-volume tables.
  • Document Object Model (DOM): Organizes elements of an HTML page into a manageable object hierarchy.
  • Expression Interpretation: Parses algorithmic operations neatly to compile order-of-operation execution scripts.

AVL Tree Operations

1. Searching Operation

Definition: A standard binary search traversal down a guaranteed balanced tree structure.

Example: Finding 25 in a tree with root 20 → 25 is larger, go right to node 30 → 25 is smaller, go left to node 25 → Match found.

Algorithm:

  1. Start at the root node.
  2. If root is empty or matches target, return current node.
  3. If target is smaller than current node, branch left.
  4. If target is larger than current node, branch right.

2. Insertion Operation

Definition: Inserts a node at a leaf slot and backtracks to apply rotations if balance is lost.

Example: Inserting 10 into path 30 → 20 makes 30 unbalanced (BF = +2) → Triggers a right rotation, pivoting 20 to the new root with children 10 and 30.

Algorithm:

  1. Traverse down to find the correct empty leaf boundary and insert the node.
  2. Backtrack up the tree path, updating the height of each ancestor node.
  3. Calculate the balance factor (Left Height - Right Height).
  4. If unbalanced, apply the matching rotation profile: LL, RR, LR, or RL.

3. Deletion Operation

Definition: Removes a target node and traces back to the root fixing cascading balance failures.

Example: Deleting node 40 with two children (35 and 50) → Swap value 40 with its inorder successor 50 → Delete the duplicate leaf node 50 → Rebalance ancestor paths.

Algorithm:

  1. Search and locate the target node.
  2. If it is a leaf, remove it. If it has one child, bypass it.
  3. If it has two children, swap with its inorder successor, then delete that successor leaf.
  4. Step backward along the path to recalculate heights and trigger corrective rotations.

1. Binary Heaps Foundation

A Binary Heap is a specialized complete binary tree structure used to achieve fast constant-time performance tracking of structural min or max key elements:

  • Structural Completeness: Perfectly fully packed across all levels from top down, filling left to right on the terminal line.
  • Max-Heap Property: Every parent node's data key is greater than or equal to its child elements.
  • Min-Heap Property: Every parent node's data key is less than or equal to its child elements.
  • Dynamic Array Indexing: For node index i, left child maps to 2i + 1, right child maps to 2i + 2, and parent node is found at floor((i - 1) / 2).

2. Applications of Heaps

Heaps provide efficient, constant-time access to maximum or minimum elements:

  • Priority Queue Engines: Coordinates multi-task execution orders where jobs are handled according to numeric weight rank parameters.
  • Heapsort: A highly stable internal sorting method providing a reliable runtime speed of O(n log n).
  • Graph Path Solvers: Speeds up shortest distance path exploration loops within Dijkstra's and Prim's algorithm scripts.

3. Graph Technical Terminology

A Graph is a non-linear network structure made up of vertices linked together by edge channels:

  • Vertex (Node): The individual structural data item container element.
  • Edge: The geometric trace path or connection link running between two adjacent vertices.
  • Directed / Undirected: Determines whether traffic flows along an explicit path direction or moves bidirectionally.
  • Path & Cycle: A path traces a sequence of unique edges; a cycle is a path that returns to its starting node.

4. Structural Representation of Graphs

Graphs are stored in memory using memory layouts tailored to the density of their edges:

  • Adjacency Matrix: A 2D array representation tracking links in O(1) time, though it demands a high O(V²) spatial allocation footprint.
  • Adjacency List: An array of linked nodes optimizing size layout down to O(V + E), making it ideal for sparse networks.

5. Graph Traversal Algorithms

Traversing means visiting all accessible vertex nodes across a network graph sequentially without falling into endless cyclic loops.

Breadth-First Search (BFS)

Explores the graph layer-by-layer, evaluating all neighbors at the current depth level before moving down into deeper branches.

  • Data Structure: Managed using an explicit FIFO Queue.
  • Use Case: Safely computes shortest paths across unweighted graphs.

Depth-First Search (DFS)

Dives straight down a single path branch as deeply as possible, backtracking up to investigate hidden side branches only when a path hits a dead end.

  • Data Structure: Implemented via Recursion or a system memory Stack.
  • Use Case: Detects cycles and maps out topological scheduling orders.
