Searching & Sorting

1. Searching Paradigms

A. Linear Search

Definition

  • Strategy: Sequential brute-force scanning strategy.
  • Execution: Evaluates items one-by-one from index 0 to N-1.
  • Prerequisite: Works seamlessly on both sorted and unsorted collections.
  • Complexity: Time: Best O(1), Worst O(n) | Space: O(1) auxiliary footprint.

Example Scenario

Finding value 7 in array [4, 2, 9, 7, 5]:
Index 0: 4 == 7 (False) → Move right.
Index 1: 2 == 7 (False) → Move right.
Index 2: 9 == 7 (False) → Move right.
Index 3: 7 == 7 (True) → Return Index 3.

Algorithm

1. Loop through index i from 0 to N-1. 2. If Array[i] == Target, return index i. 3. If the loop completes without a match, return -1.
B. Binary Search

Definition

  • Strategy: Divide-and-conquer interval lookup strategy.
  • Execution: Repeatedly halves the remaining searchable data segment.
  • Prerequisite: Input array must be sorted beforehand.
  • Complexity: Time: Best O(1), Worst O(log n) | Space: O(1) iterative space.

Example Scenario

Finding value 12 in sorted array [3, 6, 8, 12, 15, 18, 21]:
Pass 1: Low=0, High=6, Mid=3. Array[3] is 12.
Comparison: 12 == 12 (True) → Return Index 3 instantly.

Algorithm

1. Set low = 0 and high = N - 1. 2. While low <= high, calculate mid = floor((low + high) / 2). 3. If Array[mid] == Target, return mid. 4. If Target < Array[mid], narrow space by updating high = mid - 1. 5. If Target > Array[mid], narrow space by updating low = mid + 1. 6. Return -1 if loop finishes without finding a match.
2. Basic Sorting Algorithms (O(n²))
A. Bubble Sort

Definition

  • Strategy: Adjacent-comparison data sorting technique.
  • Execution: Compares neighbor indices and swaps them if out of order.
  • Behavior: Largest elements progressively drift to the end of the array.
  • Complexity: Time: Best O(n), Worst O(n²) | Space: O(1) in-place allocation.

Example Scenario

Sorting array [5, 1, 4]:
Pass 1: Compare 5 and 1 → Swap → [1, 5, 4].
Pass 2: Compare 5 and 4 → Swap →. (Sorted)

Algorithm

1. Loop i from 0 to N-1 (Outer tracking boundary). 2. Inner loop j from 0 to N-i-1. 3. If Array[j] > Array[j+1], swap their values.
B. Insertion Sort

Definition

  • Strategy: Incremental insertion sorting paradigm.
  • Execution: Divides the target collection into sorted and unsorted zones.
  • Behavior: Picks an element and shifts sorted elements right to insert it.
  • Complexity: Time: Best O(n), Worst O(n²) | Space: O(1) workspace memory.

Example Scenario

Sorting array [8, 3, 5] with item 3 as key:
Key = 3. Compare with 8 → 8 is larger, shift 8 right → [8, 8, 5].
Insert key at index 0 → [3, 8, 5].

Algorithm

1. Loop i from 1 to N-1 to select the current key = Array[i]. 2. Initialize j = i - 1. 3. While j >= 0 and Array[j] > key, shift Array[j+1] = Array[j] and decrement j. 4. Place Array[j+1] = key.
C. Selection Sort

Definition

  • Strategy: Minimum-selection boundary sort structure.
  • Execution: Scans the remaining items to locate the absolute smallest element.
  • Behavior: Swaps the identified minimum value into the current boundary slot.
  • Complexity: Time: Best/Worst O(n²) | Space: O(1) in-place allocation.

Example Scenario

Sorting array [7, 2, 4]:
Pass 1: Scan from index 0 → Minimum found is 2 → Swap with index 0 → [2, 7, 4].
Pass 2: Scan from index 1 → Minimum found is 4 → Swap with index 1 → [2, 4, 7].

Algorithm

1. Loop i from 0 to N-1. 2. Assume current minimum index min_idx = i. 3. Inner loop j from i+1 to N-1 to find smaller elements. 4. If Array[j] < Array[min_idx], update min_idx = j. 5. Swap values of Array[i] and Array[min_idx].
3. Advanced Sorting Algorithms (O(n log n))
A. Merge Sort

Definition

  • Strategy: Divide-and-conquer structural sorting method.
  • Execution: Recursively cuts arrays into equal halves until single values remain.
  • Behavior: Combines sorted halves back together using a two-pointer merge routine.
  • Complexity: Time: Best/Worst O(n log n) | Space: O(n) auxiliary buffer array.

Example Scenario

Sorting [6, 2, 8, 1]:
Split: [6, 2] and [8, 1] →, [2], [8], [1].
Merge: Combine [6] and [2] →. Combine [8] and [1] → [1, 8].
Final Merge: Combine [2, 6] and [1, 8] → [1, 2, 6, 8].

Algorithm

1. If low >= right, return immediately (base case reached). 2. Calculate mid = low + (right - low) / 2. 3. Recursively call MergeSort on the left half (low to mid). 4. Recursively call MergeSort on the right half (mid+1 to right). 5. Call Merge(low, mid, right) to combine both sections into a sorted array.
B. Quick Sort

Definition

  • Strategy: Pivot-partitioning divide-and-conquer sort layout.
  • Execution: Selects an element as a pivot to organize the surrounding data parameters.
  • Behavior: Moves smaller values left of the pivot element and larger values to the right.
  • Complexity: Time: Best O(n log n), Worst O(n²) | Space: O(log n) stack allocation.

Example Scenario

Partitioning array [4, 1, 3] with pivot value 3:
Move elements smaller than 3 to the left, larger to the right.
Rearranged array configuration maps out as: [1] → [3] → [4].

Algorithm

1. If low < high, find partition index p = Partition(low, high). 2. Recursively call QuickSort on the left sub-array (low to p-1). 3. Recursively call QuickSort on the right sub-array (p+1 to high). 4. Partition steps through data, swaps values around the pivot, and returns its final index.
C. Heap Sort

Definition

  • Strategy: Priority binary tree sorting strategy.
  • Execution: Converts the unsorted array structural elements into a max-heap structure.
  • Behavior: Repeatedly extracts the largest root node to build out the sorted index spaces.
  • Complexity: Time: Best/Worst O(n log n) | Space: O(1) in-place array sequence.

Example Scenario

Sorting array [4, 10, 3]:
Build Max-Heap → Root is 10 → Layout: [10, 4, 3].
Swap root with last item → Move 10 to end → [3, 4 | 10].
Re-heapify remaining elements → Extract 4 → [3 | 4, 10].

Algorithm

1. Build a Max-Heap from the array data (Heapify process). 2. Loop i from N-1 down to 1. 3. Swap the root element Array[0] with Array[i]. 4. Call Heapify(Array, size=i, root=0) to rebalance the remaining tree.
D. Tree Sort

Definition

  • Strategy: Binary Search Tree (BST) sorting implementation.
  • Execution: Inserts all array elements sequentially into an empty BST structure.
  • Behavior: Performs an inorder traversal loop to read the elements out in order.
  • Complexity: Time: Best O(n log n), Worst O(n²) | Space: O(n) dynamic pointer nodes.

Example Scenario

Sorting values [3, 7, 9]:
Insert 7 as root → Insert 3 to left → Insert 9 to right.
Inorder Traversal (Left, Root, Right) outputs: 3, 7, 9.

Algorithm

1. Instantiate an empty Binary Search Tree. 2. Loop through array elements and insert each value into the BST. 3. Run an Inorder Traversal function on the root node to write values back into the array.
4. Non-Comparison & External Sorting
A. Radix Sort

Definition

  • Strategy: Non-comparison integer sorting algorithm.
  • Execution: Processes numbers digit by digit, avoiding direct element-to-element comparisons.
  • Subroutine: Leverages a stable sorting algorithm backend, such as Counting Sort.
  • Complexity: Time: O(d · (n + k)) | Space: O(n + k) auxiliary working buffer space.

Example Scenario

Sorting values [142, 61, 8] by digit position:
Pass 1 (Ones Place): Sorts based on 2, 1, 8 → [61, 142, 8].
Pass 2 (Tens Place): Sorts based on 6, 4, 0 → [8, 142, 61].
Pass 3 (Hundreds Place): Sorts based on 0, 1, 0 →. (Sorted)

Algorithm

1. Find the maximum number in the array to determine the number of digits d. 2. Loop through each digit position, from least significant to most significant. 3. Run stable CountingSort on the array for the current digit position.
B. Shell Sort

Definition

  • Strategy: Interval-based structural extension of Insertion Sort.
  • Execution: Compares and adjusts elements separated by a diminishing gap sequence.
  • Behavior: Allows elements to make large jumps toward their target index coordinates early.
  • Complexity: Time: Best O(n log n), Worst O(n²) | Space: O(1) in-place footprint.

Algorithm

1. Initialize gap size gap = N / 2. 2. While gap > 0, run an interval insertion sort pass for that gap size. 3. Reduce the interval space by updating gap = gap / 2 and repeat until gap == 0.
C. External Sorting

Definition

  • Strategy: Secondary storage buffer data management paradigm.
  • Constraint: Specifically utilized when datasets are too massive to fit inside RAM.
  • Execution: Sorts data in manageable chunks and merges them back onto disk files.

Example Scenario

Sorting a massive 50GB file using only 4GB of working RAM:
Read 4GB segments into RAM, sort internally, and save them as temporary files on disk.
Read the beginning of all temporary files into RAM simultaneously to merge them into a sorted 50GB output file.

Algorithm

1. Read chunks of the large file that fit into RAM. 2. Sort each chunk using an internal method (like QuickSort) and save as a temporary run file. 3. Load the first elements of all temporary run files into memory. 4. Merge the runs together using a multi-way merge process and write the sorted output to the disk.
